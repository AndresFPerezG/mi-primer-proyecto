GIT Y GITHUB

Esta es mi propia documentación básica sobre git y github
  • GIT

    • Comandos y descripción:

    • git init: (En consola, ubicarse en la carpeta del proyecto a controlar) se inicia git.

    • git status: muestra el estado de los archivos del proyecto.

    • git add [archivo]: Agrega el o los archivos al STAGE AREA o al area de control, esto sirve para que los archivos esten listos para hacer el commit(punto de control) y también sirve para actualizar o aceptar los cambios realizados en un archivo.

    • git commit: Se crea un punto de control, esto quiere decir que podemos volver a este punto en caso de necesitar hacerlo, por ejemplo en caso de que tengamos una falla en el código y queramos volver a la configuración anterior o inicial o a la que teníamos hasta ese punto en específico.

    • git restore [archivo]: Sirve para descartar los cambios realizados a un archivo y volver al estado inicial.

    • git restore --staged [archivo]: sirve para para sacar el archivo del STAGE AREA.

    • git log: Sirve para ver todos los commits que hemos creado.
